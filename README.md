### 🌐 오픈 소스(Open Source)와 협업의 중요성

| 개념 | 설명 |
| :--- | :--- |
| **오픈 소스** | 누구나 **무료**로 코드를 **사용, 복제, 수정, 배포**할 수 있는 라이선스를 가진 소프트웨어입니다. **지식과 코드 공유**의 기반이 됩니다. |
| **GitHub** | Git 저장소를 호스팅하는 **웹 기반 플랫폼**으로, 전 세계 개발자들이 협업하고 코드를 공유하는 데 가장 널리 사용됩니다. |
| **협업/활용** | 공동 작업이 가능하며, 수많은 개발자들의 기여로 프로젝트의 **활용 가능성**과 **안정성**이 극대화됩니다. |
| **Linux** | C 언어로 구성된, **최초의 가장 성공적인 오픈 소스 운영체제** 사례입니다. |

#### 🔑 SCM (Source Code Management)의 필요성

버전 관리 시스템(SCM)은 현대 개발에서 필수적입니다.

  * **분할 작업:** 여러 명이 동시에 다른 부분을 작업할 수 있습니다.
  * **중복 작업 방지:** 작업 충돌을 효율적으로 관리합니다.
  * **오류 추적:** 언제, 누가, 무엇을 변경했는지 추적하여 문제 발생 시 쉽게 복구할 수 있습니다.

### 📝 Git 환경 설정 및 기본 명령어 가이드

Git을 사용하기 위한 필수 설정 및 **Git Bash**에서 사용되는 핵심 리눅스 명령어입니다.

#### 1\. Git 설치 및 기본 설정

1.  **Git 설치**: 로컬 환경에 Git을 설치합니다.
2.  **Git Bash 실행**: 명령줄 환경을 준비합니다.
3.  **사용자 이름/이메일 설정 (글로벌)**: 커밋 시 사용될 정보를 등록합니다. (사용자 이름은 **영문** 필수)

<!-- end list -->

```bash
git config --global user.name "kes6011"
git config --global user.email "kes6011.naver.com"
```

#### 2\. 핵심 리눅스 명령어 (Git Bash)

| 명령어 | 기능 |
| :--- | :--- |
| `clear` / `alias cl=clear` | 화면 지우기 |
| `whoami` | 현재 사용자 이름 확인 (**한글 사용 금지**) |
| `pwd` / `echo ~` | 현재 작업 디렉토리 (Home 경로) 확인 |
| `cd [경로]` | 디렉토리 이동 |
| `mkdir ./repo` | 현재 경로에 `repo` 디렉토리 생성 |
| `ls` | 파일/디렉토리 목록 출력 |
| `ls -a` | 숨김 파일을 포함한 모든 목록 출력 |
| `ls -l` | 상세 정보와 함께 목록 출력 |
| `ls -al | grep "repo"` | `repo` 문자열이 포함된 목록만 필터링 |
| `touch ./test1.txt` | 빈 파일 생성 |
| `cat ./test1.txt` | 파일 내용 출력 |
| `echo "텍스트" >> ./file.txt` | 파일 끝에 텍스트 추가 |
| `vi ./file.txt` | VI 에디터로 파일 수정 (i: 삽입, ESC + : + wq: 저장 후 종료) |
| `rm -rf ./*` | **주의\!** 현재 디렉토리의 모든 파일 및 디렉토리 삭제 |

#### 3\. SSH 키 생성 및 GitHub 등록

SSH 키는 GitHub와 안전하게 통신하기 위해 사용됩니다.

1.  **키 생성**:
    ```bash
    ssh-keygen -t ed25519 -C "GitHub 이메일"
    ```
2.  **공개 키 확인**:
    ```bash
    cd .ssh/
    cat id_ed25519.pub
    ```
3.  **GitHub 등록**: 출력된 `.pub` 파일의 내용을 **GitHub 프로필 → Settings → SSH and GPG keys → New SSH key**에 등록합니다.

### 🔄 Git 기본 워크플로우

Git을 이용한 작업은 다음 네 가지 핵심 단계로 이루어집니다.

| 순서 | 명령어 | 역할 |
| :--- | :--- | :--- |
| **1. 복제** | `git clone [SSH 주소]` | 원격 저장소의 코드를 로컬로 가져옵니다. |
| **2. 동기화 (출근)** | **`git pull`** | 원격 저장소의 최신 변경 사항을 로컬 프로젝트에 **반영**합니다. (매우 중요\!) |
| **3. 준비** | `git add .` | 변경된 파일을 **커밋**할 준비 영역(Staging Area)에 추가합니다. |
| **4. 기록** | `git commit -m "메시지"` | 변경 사항을 로컬 저장소에 **확정**하고 기록합니다. |
| **5. 공유 (퇴근)** | **`git push`** | 로컬 커밋 내용을 **원격 저장소(GitHub)에 업로드**하여 공유합니다. |

**🚨 핵심 원칙:** \*\*출근 시 `pull`, 퇴근 시 `push`\*\*를 습관화해야 충돌을 최소화할 수 있습니다.

### 💻 소스트리 (Sourcetree)를 활용한 시각화

**Sourcetree**는 Git 명령을 GUI 환경에서 직관적으로 사용할 수 있도록 돕는 도구입니다.

  * **SSH 키 등록**: 도구 → 옵션에서 생성된 SSH 키를 등록하여 GitHub 계정 인증에 사용합니다.
  * **시각화**: 변경 사항, 브랜치, 커밋 히스토리를 시각적으로 확인하여 복잡한 협업 상황을 쉽게 이해할 수 있습니다.

### 💡 용어 정리

  * **CPU (Central Processing Unit)**: 일반적으로 **단일(Single) 작업**을 순차적으로 빠르게 처리하는 데 특화되어 있습니다.
  * **GPU (Graphics Processing Unit)**: **병렬(Parallel) 연산**에 특화되어 있어, 대규모 데이터를 동시에 처리할 때 효율적입니다.
  * **Must-Have vs. Nice-to-Have**:
      * **Must-Have**: 프로젝트 성공을 위해 **반드시** 필요한 핵심 기능.
      * **Nice-to-Have**: 있다면 좋지만, 없어도 프로젝트 운영에는 지장이 없는 부가적이거나 개선할 수 있는 기능.
